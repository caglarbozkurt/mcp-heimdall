## Heimdall 0.4.0 ‚Äî Python coverage

Heimdall now covers **Python** MCP servers, and the behavioral validation from 0.3.0 already paid off ‚Äî recall is up.

### üêç New: Python coverage

Capability, provenance, dependency, and CVE analysis now cover Python servers, not just JS/TS:

```
heimdall pypi:some-mcp-server            # resolve + scan a PyPI package
heimdall ./my-python-server              # or a local directory
```

- **Capability detection** over `.py` ‚Äî `subprocess`/`os.system` ‚Üí exec, `requests`/`httpx`/`aiohttp` ‚Üí network, `os.environ` ‚Üí env, `eval`/`exec`/`compile` ‚Üí dynamic-eval, credential paths ‚Üí secret-access.
- **Provenance** from `pyproject.toml` / `requirements.txt` / `setup.py` ‚Äî a dangerous `setup.py` gates like an npm install script.
- **CVE lookups** via OSV's PyPI ecosystem (`--online`).
- Injection was already language-agnostic. Proven taint stays JS/TS-only ‚Äî Python falls back to a conservative capability co-presence gate.

### üìà Recall: 75.8% ‚Üí 80.9%

0.3.0 shipped `heimdall validate` and an honest 75.8% recall, with the misses turned into a fix-list. This release works through it:

- **Wider capability detection** ‚Äî expanded the dependency‚Üícapability map with service SDKs and HTTP clients (`googleapis`, `google-auth-library`, `isomorphic-fetch`, `@mendable/firecrawl-js`, ‚Ä¶), added family/scoped matching (`@aws-sdk/*`, `@google-cloud/*`, `@octokit/*`, and the scoped `@github/keytar` fork the exact-name map was missing), and added `vm` (`runInContext` / `compileFunction` / `new vm.Script`) to dynamic-eval detection.

Re-running the 200-package behavioral validation: **static analysis flagged 80.9% (55/68) of the capabilities servers actually exercised at runtime ‚Äî up from 75.8%**. Network misses roughly halved. The misses that remain are structural ‚Äî a capability exercised inside a dependency's internals or a subprocess, which static analysis can't see. That's the point of `validate`. Full run: [`benchmarks/validate-run.md`](https://github.com/caglarbozkurt/mcp-heimdall/blob/main/benchmarks/validate-run.md).

### Note

Python is covered *statically* here; it isn't yet covered *behaviorally* ‚Äî the `validate` recorder hooks Node, not `uvx`/`python`. That's a tracked follow-up.
